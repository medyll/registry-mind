## Supported Actions

| Action | Purpose |
|--------|---------|
| `ACTION_SIGNAL` | Desktop notification to user |
| `ACTION_SYNC` | Trigger one-time cache sync |
| `ACTION_FLUSH` | Force flush all pending packets |

## Integration Points

**OpenClaw Desktop → Mobile:**
```json
{
  "action": "com.registry.mind.ACTION_SIGNAL",
  "extras": {
    "message_type": "notification",
    "content": "Sync completed successfully"
  }
}
```

**Mobile Response:**
- Toast notification displayed
- SyncWorker enqueued for pending packets
